# 滚动回测指南

## 什么是滚动回测？

想象你在参加一门课程。老师不会只在期末考一次试，而是每个月都考一次。为什么？

- 如果只考期末一次，可能你只是运气好碰到了会的题目
- 多次考试能真实反映你的学习水平
- 每次考试后可以调整学习策略

滚动回测（Rolling Backtest）就是这个道理：
- 不是用整个历史数据回测一次就完事
- 而是分成多个时间窗口，每个窗口都独立训练、预测、回测
- 最后汇总所有窗口的结果

### 为什么普通回测不够？

假设你在 2010-2020 年的数据上训练了一个模型，回测效果很好。但这可能只是因为：
- 这个时期的某种特征特别有效
- 模型恰好"记住"了这个时期的规律
- 换到 2021-2023 年，模型可能就失效了

这就是"过拟合某个时段"。滚动回测可以避免这个问题。

## Walk-Forward 方法详解

### 时间窗口划分

滚动回测使用 Walk-Forward 方法，把时间分成多个滑动窗口：

```
时间轴: 2010 --- 2014 --- 2015 --- 2016 --- 2017 --- 2018 --- ... --- 2024

窗口 0: [TRAIN 2010-2014] [VALID 2015] [TEST 2016]
窗口 1: [TRAIN 2011-2015] [VALID 2016] [TEST 2017]
窗口 2: [TRAIN 2012-2016] [VALID 2017] [TEST 2018]
窗口 3: [TRAIN 2013-2017] [VALID 2018] [TEST 2019]
...
```

每个窗口包含三部分：

| 部分 | 用途 | 类比 |
|------|------|------|
| TRAIN（训练集） | 训练模型 | 平时练习 |
| VALID（验证集） | 调整参数 | 模拟考试 |
| TEST（测试集） | 评估最终效果 | 正式考试 |

### 每个窗口独立训练、预测、回测

以窗口 0 为例：
1. **训练**：用 2010-2014 年的数据训练模型
2. **验证**：用 2015 年的数据调整超参数
3. **预测**：用 2016 年的数据生成预测
4. **回测**：基于 2016 年的预测进行模拟交易
5. **记录**：记录 IC、Sharpe、最大回撤等指标

然后滑动到窗口 1，重复整个过程。

## 为什么要用验证集？

### 防止过拟合

想象你在准备期末考试：
- 如果你只做期末的真题，可能只是记住了答案
- 真正考试时，题目稍微一变就不会了

在机器学习中：
- 如果只用训练集，模型可能"死记硬背"了训练数据
- 在新数据上表现很差

### 用"模拟考"类比

验证集就像模拟考试：
- 模拟考的题型和正式考试类似，但题目不同
- 通过模拟考找到薄弱环节
- 调整学习方法后，再参加正式考试

在量化交易中：
- 验证集用来调整模型的超参数（比如树的深度、学习率）
- 找到最佳参数后，再在测试集上评估
- 确保测试集的结果不是"碰巧"的

---

## 滚动回测 vs 普通回测

| 对比项 | 普通回测 | 滚动回测 |
|--------|---------|---------|
| 训练数据 | 固定一个时间段 | 每个窗口不同 |
| 测试数据 | 固定一个时间段 | 每个窗口不同 |
| 结果数量 | 一个结果 | 多个结果（每个窗口一个） |
| 过拟合风险 | 高（可能过拟合特定时期） | 低（覆盖多个时期） |
| 计算成本 | 低 | 高（训练多次） |
| 可靠性 | 一般 | 高 |

### 什么时候用普通回测？

- 快速验证想法
- 计算资源有限
- 只关心特定时间段的表现

### 什么时候用滚动回测？

- 准备实盘前必须验证
- 需要全面评估策略
- 需要知道策略在不同时期的表现

## 如何运行

### 基本命令

```bash
# 运行 LightGBM 的滚动回测
uv run python scripts/roll_backtest.py --model-type lightgbm

# 运行 Kronos 的滚动回测（Kronos 只做推理，不重新训练）
uv run python scripts/roll_backtest.py --model-type kronos
```

### 自定义窗口大小

默认配置：
- 训练集：5 年
- 验证集：1 年
- 测试集：1 年
- 步长：1 年（每次滑动 1 年）

可以在代码中修改：

```python
from big_a.backtest.rolling import RollingBacktest

backtest = RollingBacktest(
    train_years=5,    # 训练年数
    valid_years=1,    # 验证年数
    test_years=1,     # 测试年数
    step_years=1,     # 滑动步长
)
```

### 默认时间范围

默认从 2010 年到 2024 年，大约生成 12 个窗口：

```
2010-2024 (15年)
- 2010-2014 训练, 2015 验证, 2016 测试 → 窗口 0
- 2011-2015 训练, 2016 验证, 2017 测试 → 窗口 1
- 2012-2016 训练, 2017 验证, 2018 测试 → 窗口 2
- ...
- 2020-2022 训练, 2023 验证, 2024 测试 → 窗口 10
```

## 结果解读

### summary_df 每行是一个窗口

运行完成后，会生成一个汇总表格 `summary_df`，每行代表一个窗口：

| window_id | train_start | train_end | test_start | test_end | ic | rank_ic | icir | sharpe | max_drawdown |
|-----------|-------------|-----------|------------|----------|----|---------|------|--------|--------------|
| 0 | 2010-01-01 | 2014-12-31 | 2016-01-01 | 2016-12-31 | 0.045 | 0.062 | 0.89 | 1.2 | 0.15 |
| 1 | 2011-01-01 | 2015-12-31 | 2017-01-01 | 2017-12-31 | 0.038 | 0.055 | 0.75 | 1.0 | 0.18 |
| 2 | 2012-01-01 | 2016-12-31 | 2018-01-01 | 2018-12-31 | 0.052 | 0.070 | 0.95 | 1.4 | 0.12 |
| ... | ... | ... | ... | ... | ... | ... | ... | ... | ... |

### mean_ic, mean_sharpe 等是所有窗口的平均值

除了每个窗口的详细指标，还会计算所有窗口的统计量：

```python
# 所有窗口的平均 IC
mean_ic = summary_df['ic'].mean()

# 所有窗口的最小 Sharpe（最坏情况）
min_sharpe = summary_df['sharpe'].min()

# IC 的标准差（稳定性）
std_ic = summary_df['ic'].std()
```

### 如何判断滚动回测结果？

好的滚动回测结果应该满足：
1. **大部分窗口的 IC 都 >= 0.03**：不是只有少数窗口表现好
2. **mean_sharpe >= 1.0**：整体收益匹配风险
3. **min_sharpe > 0.5**：即使是最坏的窗口，也不至于太差
4. **IC 的标准差较小**：表现稳定，不是忽好忽坏

### MLflow 跟踪

每个窗口的结果都会自动记录到 MLflow 中，包括：
- 模型参数
- 评估指标（IC、Sharpe 等）
- 训练时间
- 模型文件

这样你可以：
- 对比不同参数的效果
- 追踪模型改进的历史
- 复现最好的结果

## 常见问题

### Q: 为什么 Kronos 不需要重新训练？

A: Kronos 是一个预训练的大语言模型，它的参数已经固定。在滚动回测中，Kronos 只做推理，不需要在每个窗口重新训练。

### Q: 窗口大小怎么设置？

A: 需要根据市场特点和策略类型调整：
- **快速变化的市场**：训练集短一些（比如 3 年），更及时适应新环境
- **慢速变化的市场**：训练集长一些（比如 7 年），充分利用历史规律
- **验证集**：至少 6 个月，才能有效调整参数
- **测试集**：至少 1 年，才能可靠评估

### Q: 窗口之间重叠吗？

A: 是的。窗口 0 的测试集是 2016 年，窗口 1 的测试集是 2017 年，等等。这样充分利用了历史数据。

### Q: 滚动回测需要多长时间？

A: 取决于数据量和模型复杂度：
- LightGBM：每个窗口训练约 5-10 分钟，12 个窗口约 1-2 小时
- Kronos：每个窗口推理约 1-2 分钟，12 个窗口约 15-30 分钟

### Q: 如何判断是否需要滚动回测？

A: 建议在以下情况使用滚动回测：
- 准备投入实盘前
- 策略参数有重大调整
- 想全面评估策略的稳健性
- 需要写报告或展示给他人

## 完整示例

```python
from big_a.backtest.rolling import RollingBacktest
from big_a.models.lgb_model import LGBModel
import pandas as pd

# 1. 准备数据
train_data = pd.read_parquet('data/train.parquet')
test_data = pd.read_parquet('data/test.parquet')

# 2. 创建滚动回测对象
backtest = RollingBacktest(
    train_years=5,
    valid_years=1,
    test_years=1,
    step_years=1,
    model_type='lightgbm',  # 或 'kronos'
)

# 3. 运行滚动回测
results = backtest.run(
    train_data=train_data,
    test_data=test_data,
)

# 4. 查看汇总结果
summary = backtest.aggregate_results(results)
print(summary)

# 5. 分析具体窗口
window_0 = results[0]
print(f"窗口 0 的 IC: {window_0.ic:.4f}")
print(f"窗口 0 的 Sharpe: {window_0.sharpe:.2f}")
print(f"窗口 0 的最大回撤: {window_0.max_drawdown:.2%}")
```

## 总结

滚动回测是量化交易的"黄金标准"，它能：
- 避免过拟合特定时期
- 评估策略在不同市场环境下的表现
- 提供更可靠的实盘预期

记住：
- 滚动回测成本高，但值得
- 不仅看平均值，还要看最坏情况
- 稳定性比偶尔的高分更重要

只有通过滚动回测验证的策略，才能真正放心地投入实盘。
