# 评估模块详解

## 为什么需要评估？

想象你在参加一场考试。考完后，老师给了你一个分数，比如 85 分。这个分数告诉你考得怎么样，但更重要的是，它让你知道自己的学习方式是否有效。

在量化交易中，评估模块就像这场考试的老师。我们的"学生"是预测模型，"考试题目"是每天的股票市场，"考试分数"就是各种评估指标。这些指标告诉我们：

- 模型预测得准不准？
- 模型赚的钱多不多？
- 模型会不会亏大钱？
- 模型的表现是不是靠运气？

只有通过全面评估，我们才能判断一个模型是否值得实盘使用。

## IC（信息系数）详解

### 什么是相关性？

在生活中，很多事物之间有关联。比如，身高和体重通常正相关：个子高的人往往体重也重。如果我们统计一群人，画一个散点图，你会看到点大致沿着一条线向右上方延伸。

数学上，我们用"相关系数"来衡量这种关联的强弱：
- 相关系数在 -1 到 1 之间
- 1 表示完全正相关（A 增加时 B 也增加）
- -1 表示完全负相关（A 增加时 B 减少）
- 0 表示没有关系

### IC = 预测分数和实际收益的相关性

在量化交易中：
- 预测分数：模型认为某只股票应该涨多少
- 实际收益：这只股票第二天实际涨了多少

IC 就是这两组数据的相关系数。计算方法是：

1. 对某一天的所有股票，计算预测分数
2. 记录这些股票当天的实际收益
3. 计算这两组数据的皮尔逊相关系数

### IC > 0 表示预测方向正确

如果 IC = 0.05，这说明：
- 模型预测高的股票，确实倾向于涨得更多
- 预测方向大体正确
- 但预测数值和实际数值可能不完全一致

如果 IC = -0.02，这说明：
- 模型预测高的股票，反而倾向于涨得少
- 预测方向反了
- 需要调整模型或因子

### 为什么 IC >= 0.03 就算有用？

在 A 股市场，0.03 已经是一个非常不错的 IC 值了。这是因为：

- 股票市场充满了噪音，预测很难
- 0.03 的 IC 意味着每天能从市场中提取一点点"规律"
- 虽然看起来很小，但累积起来就很可观了

类比一下：
- 如果你能连续猜对 51% 的彩票号码，已经能赚钱了
- IC = 0.03 的效果就像 51% 的胜率一样

**成功阈值**：IC >= 0.03

## Rank IC

### 和 IC 的区别

IC 计算数值的相关性，而 Rank IC 只看排名。

举个例子：
- 股票 A：预测 10%，实际 8%
- 股票 B：预测 5%，实际 6%
- 股票 C：预测 2%，实际 3%

对于 IC，我们关心的是 10%、5%、2% 和 8%、6%、3% 的数值相关性。

对于 Rank IC，我们只关心排名：
- 预测排名：A > B > C
- 实际排名：A > B > C
- 排名完全一致，Rank IC = 1

### 为什么更稳健？

Rank IC 使用斯皮尔曼秩相关系数，它对极端值不敏感。

假设某一天有一只股票暴涨 20%，而其他股票都在 ±3% 内波动：
- IC 会被这只极端股票影响很大
- Rank IC 只看排名，不受极端值影响

因此在实践中，Rank IC 往往比 IC 更稳定。

## ICIR

### IC 的"稳定性"

ICIR = mean(IC) / std(IC)

公式看起来复杂，但含义很简单：
- 分子：平均 IC（预测能力的平均水平）
- 分母：IC 的标准差（预测能力的波动程度）

ICIR 就像 IC 的"信噪比"。

### 高 ICIR = 预测持续有效，不是偶尔蒙对

- ICIR 高：模型每天都能稳定预测，不是靠运气
- ICIR 低：模型表现忽好忽坏，可能是运气

类比：
- 考生 A：每次考试都在 85-90 分之间，ICIR 高
- 考生 B：有时考 100 分，有时考 40 分，ICIR 低

虽然两人平均分可能一样，但考生 A 更可靠。

## Sharpe 比率

### "每承受 1 单位风险，能获得多少收益"

Sharpe Ratio = sqrt(252) * mean(日收益) / std(日收益)

这个指标回答了一个关键问题：**承担这个风险值得吗？**

- 分子：年化收益（乘以 sqrt(252) 是为了从日收益转为年化）
- 分母：收益的波动（风险）

Sharpe 比率越高，说明每单位风险带来的收益越大。

### Sharpe > 1 算不错，> 2 算优秀

- Sharpe < 1：风险大，收益相对小，不太划算
- Sharpe = 1-2：还不错，风险和收益匹配
- Sharpe > 2：非常优秀，收益远超风险

类比：
- 工作 A：月薪 1 万，每天加班到深夜（低 Sharpe）
- 工作 B：月薪 2 万，工作轻松（高 Sharpe）

**成功阈值**：Sharpe >= 1.0

## 最大回撤（Max Drawdown）

### 从最高点跌到最低点的幅度

Max Drawdown = max(最高点 - 最低点) / 最高点

假设你的投资曲线是这样的：
```
净值
  ^
  |        ★ 最高点
  |       / \
  |      /   \
  |     /     \ ★ 最低点
  |____/_______\________ 时间
```

从最高点 100% 跌到最低点 80%，最大回撤就是 20%。

### 用"从山顶到山谷"类比

想象你在爬山：
- 爬到山顶（净值最高点）
- 突然滚落到山谷（净值最低点）
- 最大回撤就是你滚落的高度

### 为什么重要：回撤太大可能导致"扛不住"

心理层面：
- 回撤 10%：还能接受
- 回撤 20%：开始慌了
- 回撤 50%：很多人会割肉止损

实际层面：
- 回撤太大，可能触发风控强制平仓
- 资金大幅缩水，很难再赚回来（亏 50% 需要涨 100% 才能回本）

**成功阈值**：Max Drawdown <= 20%

## 信息比率（Information Ratio）

### 超额收益的稳定性

Information Ratio = sqrt(252) * mean(超额收益) / std(超额收益)

超额收益 = 投资组合收益 - 基准收益（比如沪深 300）

信息比率衡量的是：**你比市场多赚的钱，是不是稳定的？**

### 和 Sharpe 的区别

| 指标 | 衡量内容 | 用途 |
|------|---------|------|
| Sharpe Ratio | 绝对收益的稳定性 | 判断策略是否值得做 |
| Information Ratio | 超额收益的稳定性 | 判断是否跑赢市场 |

Sharpe 关心"我赚得够不够"，Information Ratio 关心"我比别人赚得够不够多"。

## 换手率（Turnover）

### 每天调仓了多少比例

Turnover = (买入金额 + 卖出金额) / 2 / 总资产

如果今天把组合里的股票全部换一遍，换手率就是 100%。

### 高换手率 = 高交易成本

- 交易成本：佣金 + 印花税 + 滑点
- A 股单边成本大约 0.1%-0.3%
- 如果每天换手率 100%，一年 250 个交易日，成本就是 25%-75%

这就是为什么高换手策略很难赚钱。

换手率应该平衡：
- 太低：可能错过机会
- 太高：交易成本吃掉利润

## 模型对比

### compare_models() 如何工作

当你有多个模型时，需要知道哪个更好。`compare_models()` 函数会：

1. 收集每个模型的预测结果
2. 对每个模型计算 IC、Rank IC、ICIR
3. 整理成表格，方便对比

### 如何解读对比结果

结果表格大概长这样：

| 模型 | IC | Rank IC | ICIR |
|------|----|---------|------|
| LightGBM | 0.045 | 0.062 | 0.89 |
| Kronos | 0.038 | 0.055 | 0.75 |
| 随机森林 | 0.020 | 0.030 | 0.40 |

从这张表可以看出：
- LightGBM 的 IC 最高（0.045），预测能力最强
- LightGBM 的 ICIR 最高（0.89），最稳定
- 随机森林表现最差，不建议使用

## 代码示例

```python
from big_a.evaluation.metrics import compute_ic, compute_icir, compute_sharpe
from big_a.evaluation.metrics import compare_models, WindowResult

# 1. 计算 IC
# predictions: 模型预测值，shape: [日期, 股票]
# returns: 实际收益，shape: [日期, 股票]
ic = compute_ic(predictions, returns)
print(f"IC: {ic.mean():.4f}")

# 2. 计算 ICIR
icir = compute_icir(ic)
print(f"ICIR: {icir:.4f}")

# 3. 计算 Sharpe 比率
daily_returns = ...  # 每日收益
sharpe = compute_sharpe(daily_returns)
print(f"Sharpe: {sharpe:.2f}")

# 4. 对比多个模型
model_predictions = {
    'lightgbm': lgb_predictions,
    'kronos': kronos_predictions,
}
comparison = compare_models(model_predictions, returns)
print(comparison)

# 5. 滚动回测的窗口结果
window_result = WindowResult(
    ic=0.045,
    rank_ic=0.062,
    icir=0.89,
    sharpe=1.2,
    max_drawdown=0.15,
)
print(f"这个窗口的 IC 是 {window_result.ic:.3f}")
```

## 总结

评估模块是量化交易的核心，它回答了一个根本问题：**这个策略真的能赚钱吗？**

记住这些关键指标：
- **IC >= 0.03**：预测方向正确
- **Sharpe >= 1.0**：收益匹配风险
- **Max Drawdown <= 20%**：亏得不会太惨
- **高 ICIR**：表现稳定，不是运气

只有满足这些条件的策略，才值得投入实盘。
