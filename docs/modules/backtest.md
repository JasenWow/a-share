# 回测引擎详解

## 1. 什么是回测？

想象你有一台时光机，可以回到过去，带着现在的策略去投资。

比如现在是 2024 年，你想测试一个策略："每天买入过去 5 天涨幅最大的 10 只股票"。

你用时光机回到 2015 年，按照这个规则每天交易，记录每一笔买卖，计算最终收益。这就是回测。

回测的本质是：**用历史数据模拟交易，评估策略的有效性**。

如果没有回测，你只能用真金白银去试验，亏了钱才能知道策略不行。有了回测，可以在不亏钱的情况下测试无数种策略。

## 2. 回测的基本原理

回测就像玩一个策略游戏，每天按规则操作：

### 第一步：生成信号

模型每天对所有股票打分，选出"最看好"的股票。

比如模型给 5000 只股票打分：
- 000001.SZ: 0.05（预测涨 5%）
- 000002.SZ: 0.03（预测涨 3%）
- ...
- 600000.SH: -0.02（预测跌 2%）

### 第二步：选股

根据策略规则选股。Big-A 使用 TopkDropout 策略：

- TopK = 50：选分数最高的 50 只股票
- Dropout = 5：每天淘汰表现最差的 5 只

### 第三步：执行交易

每天收盘后：
1. 卖掉需要淘汰的 5 只股票
2. 买入新选中的 5 只股票
3. 按收盘价成交
4. 扣除交易成本

### 第四步：记录收益

每天计算：
- 今天持仓的市值变化
- 今天的交易成本
- 今天的总收益

累积得到整个回测期间的收益曲线。

## 3. A 股回测的特殊规则

A 股有很多特殊规则，回测时必须考虑，否则结果会严重失真。

### 3.1 T+1 制度的影响

**规则：** 今天买的股票，明天才能卖。

**为什么重要？**

假设你在 1 月 1 日收盘前买入股票，当天股价涨了 5%。因为 T+1，你只能在 1 月 2 日及以后卖出，无法锁定 1 月 1 日的 5% 收益。

如果 1 月 2 日股价跌了 10%，你反而亏了 5%。

**回测如何处理？**

Big-A 的标签设计考虑了 T+1：
- 预测的不是明天的收益率
- 而是后天的收益率（`Ref($close, -2) / Ref($close, -1) - 1`）
- 这样就能正确反映 T+1 的影响

### 3.2 涨跌停限制

**规则：** 每天涨跌幅限制在 ±10%（科创板和创业板是 ±20%）。

**为什么重要？**

假设模型预测某只股票明天会涨 15%，但实际上明天只能涨到涨停板 10%，预测就被"截断"了。

**回测如何处理？**

Big-A 设置 `limit_threshold = 0.095`（9.5%）：

- 如果预测价格超过昨天的 9.5%，按涨停价成交
- 如果预测价格低于昨天的 9.5%，按跌停价成交
- 用 9.5% 而不是 10%，是留一点缓冲空间

### 3.3 交易成本详解

A 股交易成本包括三部分：

**1. 佣金（手续费）**

- 买入：0.025%（万分之 2.5）
- 卖出：0.025%（万分之 2.5）
- 最低 5 元

比如买入 10000 元股票：
- 佣金 = 10000 × 0.00025 = 2.5 元
- 因为低于 5 元，按 5 元收取

**2. 印花税**

- 只在卖出时收取
- 费率：0.1%（千分之一）

比如卖出 10000 元股票：
- 印花税 = 10000 × 0.001 = 10 元

**3. 过户费**

- 买入和卖出都收取
- 费率：0.001%（万分之一）

比如买卖 10000 元股票：
- 过户费 = 10000 × 0.0001 = 1 元（买）+ 1 元（卖）= 2 元

**总结：**

- 买入总成本 = 佣金（5 元起） + 过户费
- 卖出总成本 = 佣金（5 元起） + 印花税 + 过户费
- Big-A 设置：
  - `open_cost = 0.0005`（买入 0.05%，包含佣金和过户费）
  - `close_cost = 0.0015`（卖出 0.15%，包含佣金、印花税和过户费）
  - `min_cost = 5`（最低 5 元）

**为什么交易成本很重要？**

假设你有一个策略：
- 不考虑成本：年化收益 20%
- 考虑成本：年化收益 15%

少了 5%！对于 1 亿资金，就是 500 万的差异。

如果策略频繁交易（比如每天换手 20%），成本会更高，甚至可能把盈利全部吃掉。

## 4. TopkDropout 策略详解

### 4.1 策略定义

**TopK：** 每天选模型评分最高的 K 只股票

**Dropout：** 每天淘汰表现最差的 N 只股票

Big-A 默认参数：
- `topk = 50`：持仓 50 只股票
- `n_drop = 5`：每天换 5 只股票

### 4.2 用"选秀节目"类比

想象你在办一个股票选秀节目：

**第一轮（TopK）：**

- 5000 只股票报名参加
- 模型给每只股票打分
- 选出分数最高的 50 只进入"选手池"

**第二轮（每日更新）：**

- 第二天，选手池里的 50 只股票中，表现最差的 5 只被淘汰
- 从剩下的 4950 只股票中，选出分数最高的 5 只补位
- 选手池始终保持 50 只

**第三轮（持续进行）：**

- 每天重复淘汰 5 只、补充 5 只的过程
- 优秀的股票留在池中，表现不好的被淘汰

### 4.3 为什么用 TopkDropout？

**优点：**

1. **分散风险**：持有 50 只股票，单只股票影响有限
2. **动态调整**：每天更新持仓，及时淘汰表现差的股票
3. **控制换手**：每天只换 5 只（10% 换手率），避免过度交易
4. **简单有效**：不需要复杂的仓位管理

**缺点：**

1. **等权重**：所有股票权重相同，不能根据信心调整
2. **忽略行业**：可能全仓一个行业，风险集中
3. **止损不足**：表现差的股票要等到第二天才能卖出

### 4.4 参数选择建议

| topk | n_drop | 换手率 | 适用场景 |
|------|--------|--------|----------|
| 30 | 3 | 10% | 集中投资，追求高收益 |
| 50 | 5 | 10% | 平衡收益和风险（默认） |
| 100 | 10 | 10% | 分散投资，追求稳健 |
| 50 | 10 | 20% | 快速调整，捕捉热点 |

**换手率计算：**

```
换手率 = n_drop / topk × 100%
```

比如 topk=50, n_drop=5，换手率 = 5/50 × 100% = 10%

换手率越高，交易成本越高，需要策略有足够高的超额收益来覆盖成本。

## 4.5 实盘交易策略（RealTradingStrategy）

### 4.5.1 策略定义

**RealTradingStrategy** 是在 TopkDropout 策略基础上，增加了风控机制的实盘友好型策略。

### 4.5.2 用"保险投资"类比

想象你在投资时买了一份"保险"：

**TopkDropout 就像"裸投"：**
- 每天选最好的 50 只股票
- 表现差的第二天就卖出
- 没有止损，可能一直亏下去
- 仓位没有限制，可能一只股票买太多

**RealTradingStrategy 就像"买了保险的投资"：**
- 每天选最好的 50 只股票（和 TopkDropout 一样）
- **止损机制**：任何股票亏损超过 8%，立即卖出（就像保险赔你钱，及时止损）
- **周频调仓**：不是每天调整，而是每周（5 个交易日）调整一次（降低交易成本）
- **仓位限制**：任何一只股票最多占总资产的 25%（避免"鸡蛋放在一个篮子里"）

### 4.5.3 三个核心风控机制

**1. 止损（Stop Loss）**

每天检查持仓的每只股票：
- 如果（当前价格 - 买入价格）/ 买入价格 < -8%
- 立即卖出这只股票
- 不等周频调仓日

**类比**：就像你买了一只股票，如果跌了 8%，就当"亏了"赶紧卖掉，避免跌到 20% 才卖。

**2. 周频调仓（Weekly Rebalance）**

每隔 5 个交易日（一周）才执行一次选股和调仓：
- 每天仍会检查止损
- 但选股和补位只在调仓日进行

**类比**：就像你每周一才"整理一下"你的投资组合，平时只做止损，不频繁调仓。

**3. 仓位上限（Position Weight Cap）**

任何一只股票的权重不超过 25%：
- 如果选中的 5 只股票，每只最多占 25%
- 如果需要买更多，就按 25% 限制买入，剩下的资金留作现金

**类比**：就像你规定自己"不要把超过 25% 的钱放在一只股票上"，避免单只股票暴雷。

### 4.5.4 与 TopkDropout 的区别

| 特性 | TopkDropout | RealTradingStrategy |
|------|-------------|-------------------|
| 选股机制 | 每天 | 每 5 天（周频） |
| 止损 | 无 | 有（-8%） |
| 仓位限制 | 无 | 有（最大 25%） |
| 交易成本 | 较高（每日调仓） | 较低（周频调仓） |
| 风险控制 | 较弱 | 较强 |
| 适用场景 | 回测研究 | 实盘交易 |

### 4.5.5 配置示例

配置文件：`configs/backtest/real_trading.yaml`

```yaml
strategy:
  type: real_trading
  topk: 50              # 持仓 50 只股票
  n_drop: 5             # 每次调仓换 5 只
  stop_loss: -0.08      # 止损阈值 -8%
  max_weight: 0.25      # 仓位上限 25%
  rebalance_freq: 5     # 每 5 个交易日调仓一次
```

### 4.5.6 使用方法

```bash
# 运行实盘策略回测
uv run python scripts/real_trading_backtest.py \
  --config configs/backtest/real_trading.yaml \
  --output output/real_trading_report.parquet
```

### 4.5.7 什么时候使用 RealTradingStrategy？

- **准备实盘交易**：需要在回测时就考虑实盘的风控限制
- **风险厌恶型**：希望降低最大回撤，控制单只股票风险
- **降低交易成本**：周频调仓比日频调仓成本低

### 4.5.8 注意事项

1. **止损的局限性**：如果股票连续跌停（比如直接 -10%），可能无法在 -8% 时卖出
2. **调仓频率权衡**：调仓越频繁，交易成本越高；调仓越慢，可能错过机会
3. **仓位上限的影响**：限制了高置信度股票的仓位，可能降低收益

## 5. 回测结果怎么看？

### 5.1 Report DataFrame 每一列的含义

回测生成的 report 是一个 DataFrame，记录每天的详细数据：

| 列名 | 含义 | 示例 |
|------|------|------|
| date | 日期 | 2024-01-02 |
| return | 策略当日收益率 | 0.012（涨 1.2%） |
| bench | 基准（沪深300）当日收益率 | 0.008（涨 0.8%） |
| cost | 当日交易成本 | -0.0005（亏 0.05%） |
| turnover | 当日换手率 | 0.1（换手 10%） |

**示例数据：**

```
date       return    bench     cost      turnover
2024-01-02  0.012     0.008    -0.0005   0.1
2024-01-03  0.005     0.003    -0.0004   0.1
2024-01-04 -0.008    -0.005    -0.0003   0.1
```

解读：
- 2024-01-02：策略涨 1.2%，基准涨 0.8%，扣除 0.05% 成本后，净收益 1.15%
- 2024-01-04：策略跌 0.8%，基准跌 0.5%，策略表现不如基准

### 5.2 NAV 曲线

NAV（Net Asset Value，净值）曲线展示 1 元钱随时间的变化。

**计算公式：**

```
NAV_t = NAV_{t-1} × (1 + return_t)
```

**示例：**

| 日期 | 当日收益 | NAV |
|------|----------|-----|
| 初始 | - | 1.000 |
| 第一天 | 0.01（+1%） | 1.010 |
| 第二天 | -0.005（-0.5%） | 1.005 |
| 第三天 | 0.02（+2%） | 1.025 |

**如何解读 NAV 曲线：**

- 曲线向上：策略赚钱
- 曲线向下：策略亏钱
- 曲线波动大：风险高
- 曲线平滑：风险低

**常见问题：**

Q: NAV 从 1.0 涨到 2.0，意味着什么？
A: 意味着 1 元钱变成了 2 元，收益率 100%。

Q: 为什么 NAV 曲线和基准曲线的起点不同？
A: 因为初始资金可能不同，或者基准的起点时间不同。重要的是看两条曲线的相对表现。

### 5.3 回撤

回撤（Drawdown）是从历史最高点到当前点的跌幅。

**计算公式：**

```
回撤 = (当前 NAV - 历史最高 NAV) / 历史最高 NAV
```

**示例：**

| 日期 | NAV | 历史最高 | 回撤 |
|------|-----|----------|------|
| 第一天 | 1.10 | 1.10 | 0% |
| 第二天 | 1.15 | 1.15 | 0% |
| 第三天 | 1.05 | 1.15 | -8.7% |
| 第四天 | 1.20 | 1.20 | 0% |

解读：
- 第三天回撤 8.7%，意味着从最高点 1.15 跌到了 1.05，跌幅 (1.05 - 1.15) / 1.15 = -8.7%

**最大回撤（Max Drawdown）：**

整个回测期间最大的回撤值。比如最大回撤是 -20%，意味着在最糟糕的时候，从最高点跌了 20%。

**回撤持续时间：**

从创新高到回到前高的时间。比如：
- 1 月 1 日创新高（NAV = 1.20）
- 1 月 2 日到 1 月 15 日都在 1.20 以下
- 1 月 16 日再次突破 1.20
- 回撤持续时间 = 14 天

**如何评估回撤？**

- 最大回撤 < 10%：风险低
- 最大回撤 10% - 20%：风险中等
- 最大回撤 > 20%：风险高

但要结合收益看：
- 年化 20%，最大回撤 10% → 优秀
- 年化 5%，最大回撤 10% → 一般

## 7. 分析报告的指标解释

### 7.1 年化收益率（Annualized Return）

**定义：** 把回测期间的总收益折算成一年的收益。

**计算公式：**

```
年化收益率 = (1 + 总收益率)^(252 / 交易天数) - 1
```

假设：
- 回测 2 年（504 个交易日）
- 总收益率 50%
- 年化收益率 = (1 + 0.5)^(252 / 504) - 1 = 1.5^0.5 - 1 = 22.47%

**如何评估？**

- 年化 > 20%：优秀
- 年化 10% - 20%：良好
- 年化 5% - 10%：一般
- 年化 < 5%：较差

但要考虑风险和市场环境。

### 7.2 夏普比率（Sharpe Ratio）

**定义：** 每承担一单位风险，能获得多少超额收益。

**计算公式：**

```
夏普比率 = sqrt(252) × (策略平均收益 - 无风险利率) / 策略收益标准差
```

假设：
- 策略日均收益 0.08%
- 无风险利率 0.02%（日均）
- 策略日收益标准差 1.5%
- 夏普比率 = sqrt(252) × (0.0008 - 0.0002) / 0.015 = 15.87 × 0.0006 / 0.015 = 0.63

**如何评估？**

- 夏普 > 2：优秀
- 夏普 1 - 2：良好
- 夏普 0.5 - 1：一般
- 夏普 < 0.5：较差

夏普比率越高越好，但要注意：
- 夏普比率对极端值敏感
- 短期回测的夏普比率可能失真

### 7.3 信息比率（Information Ratio）

**定义：** 策略相对于基准的超额收益与跟踪误差的比值。

**计算公式：**

```
信息比率 = (策略收益 - 基准收益)的均值 / (策略收益 - 基准收益)的标准差
```

假设：
- 策略年化收益 15%
- 基准年化收益 10%
- 超额收益年化标准差 8%
- 信息比率 = (0.15 - 0.10) / 0.08 = 0.625

**如何评估？**

- 信息比率 > 1：优秀
- 信息比率 0.5 - 1：良好
- 信息比率 < 0.5：一般

信息比率衡量的是"跑赢基准的能力"。

### 7.4 最大回撤（Max Drawdown）

前面已经详细解释，这里补充评估标准：

- 最大回撤 < 10%：低风险
- 最大回撤 10% - 20%：中等风险
- 最大回撤 > 20%：高风险

但要结合收益看：

| 年化收益 | 最大回撤 | 评价 |
|----------|----------|------|
| 20% | 10% | 优秀 |
| 20% | 30% | 风险较高 |
| 5% | 10% | 收益风险比不理想 |
| 5% | 5% | 非常稳健 |

## 8. 可视化图表说明

### 8.1 NAV 曲线图

**用途：** 展示策略和基准的净值变化。

**解读：**
- 策略曲线在基准曲线上方：策略跑赢基准
- 策略曲线斜率更大：策略增长更快
- 策略曲线波动更大：策略风险更高

### 8.2 回撤图

**用途：** 展示回撤随时间的变化。

**解读：**
- 回撤越深：风险越大
- 回撤持续时间越长：恢复越慢
- 多次大幅回撤：策略不稳定

### 8.3 月度收益热力图

**用途：** 展示每个月的收益情况。

**颜色：**
- 红色：正收益
- 蓝色：负收益
- 颜色越深：收益/亏损越大

**解读：**
- 哪些月份表现好/差
- 是否有季节性规律
- 是否连续多个月亏损

### 8.4 IC 系列（Information Coefficient）

**用途：** 衡量预测值和真实值的相关性。

**计算：** 每天计算预测排名和真实收益排名的斯皮尔曼相关系数。

**解读：**
- IC > 0：预测和真实正相关（预测对了）
- IC < 0：预测和真实负相关（预测反了）
- IC = 0：预测和真实无关（没预测准）

**IC 均值 > 0.05，IC 标准差 < 0.15：** 预测能力较好。

## 9. 常见误区

### 9.1 过拟合（Overfitting）

**什么是过拟合？**

模型在历史数据上表现很好，但在未来实盘中表现很差。

**原因：**

模型"记住了"历史数据，而不是"学会"了规律。就像学生死记硬背，考试换道题就不会了。

**例子：**

- 模型发现："2015 年 6 月 15 日买入 000001.SZ 必涨"
- 这是特定时间、特定股票的巧合，不是普遍规律
- 未来不会重复这个巧合

**如何避免？**

1. **样本外测试**：用一部分数据不参与训练，只做测试
2. **交叉验证**：多次划分训练集和测试集
3. **简化模型**：不要用太复杂的模型，容易过拟合
4. **参数调优**：用验证集调参，而不是用测试集

### 9.2 未来函数（Look-ahead Bias）

**什么是未来函数？**

在回测时使用了当时还不知道的数据。

**例子：**

- 用收盘后的数据判断当天是否应该买入（应该在收盘前判断）
- 用当天的财务数据判断当天是否应该买入（财务数据通常滞后）
- 用未来几天的数据调整当天的仓位（这是作弊！）

**后果：**

回测结果虚高，实盘无法复现。

**如何避免？**

1. **严格按时间顺序**：只使用截至当时的数据
2. **了解数据发布时间**：财务数据、宏观数据都有发布延迟
3. **代码审查**：检查是否意外使用了未来数据

### 9.3 幸存者偏差（Survivorship Bias）

**什么是幸存者偏差？**

只统计现存的公司，忽略了已经退市的公司。

**例子：**

回测 2010-2020 年的 A 股，如果只用现在还在上市的 3000 只股票：

- 忽略了这期间退市的几百家公司
- 这些退市公司通常表现很差
- 回测结果被高估

**后果：**

高估策略收益，低估风险。

**如何避免？**

1. **使用完整的历史股票池**：包含已经退市的公司
2. **数据源要可靠**：Qlib 等专业数据源已经处理了这个问题
3. **定期检查**：确认股票池的完整性

### 9.4 其他常见问题

**1. 忽略交易成本：**

- 回测收益 20%，扣除成本后可能只有 15%
- 频繁交易的策略尤其要注意

**2. 忽略流动性：**

- 回测假设可以按收盘价买卖任意数量
- 实际上，小盘股可能买不到或卖不掉

**3. 忽略滑点：**

- 回测假设按理论价格成交
- 实际可能有滑点（买入价更高，卖出价更低）

**4. 回测时间太短：**

- 1 年的回测可能只是运气好
- 建议至少 3-5 年，包含牛市、熊市、震荡市

**5. 数据挖掘：**

- 尝试 100 种策略，选表现最好的 1 种
- 这只是运气，不是策略真的好
- 需要在样本外验证

## 10. 总结

回测是量化交易的关键环节，就像考试前的模拟测试。

**要点回顾：**

1. **回测原理**：用历史数据模拟交易，评估策略
2. **A 股特殊规则**：T+1、涨跌停、交易成本必须考虑
3. **TopkDropout 策略**：分散风险、动态调整
4. **回测结果解读**：NAV、回撤、夏普比率等指标
5. **常见误区**：过拟合、未来函数、幸存者偏差

**记住：**

- 回测好不代表实盘好
- 回测差通常实盘也差
- 回测是必要条件，不是充分条件
- 实盘前必须严格回测，但回测后仍需谨慎

**最终建议：**

- 用回测筛选策略
- 用小资金验证策略
- 逐步放大资金规模
- 持续监控和优化

回测是工具，不是保证。投资有风险，入市需谨慎。

## 11. 小资金实盘策略（RealTradingStrategy）

RealTradingStrategy 是专门为小资金（约100万）设计的实盘交易策略，相比 TopkDropout 更加保守，增加了风险控制机制。

### 11.1 为什么小资金需要不同策略？

大资金（如1亿）可以持有50只股票，分散风险。但小资金（100万）如果也持有50只股票，每只股票平均只有2万元，交易成本占比很高。

RealTradingStrategy 的设计原则：
- **更少的持仓**：只持有 4-5 只股票，每只股票约 20-25 万元
- **更低的换手**：每周调仓一次，减少交易成本
- **更严格的风控**：止损、仓位上限，防止大额亏损

### 11.2 三大风险控制机制

#### 止损（Stop Loss）

**机制**：单只股票亏损超过 8% 自动卖出。

**类比**：就像家里的保险丝——当电流超过安全值时自动断电，防止电器烧毁。止损就是在亏损超过你能承受的范围时自动"断电"（卖出），防止更大的损失。

**为什么重要**：小资金抗风险能力弱，一次大亏损可能就回不去了。设置 8% 止损线，可以把单只股票的最大亏损控制在总资金的 2% 以内（假设持仓 25%）。

#### 周频调仓（Weekly Rebalancing）

**机制**：每 5 个交易日才选股换仓，不是每天换。

**类比**：不是每天去超市买菜，而是每周去一次。每天买菜既累又费油，每周一次既省力又省钱。

**为什么重要**：每天换仓会产生大量交易成本。小资金每周换一次，可以节省 80% 的交易成本，同时还能捕捉到中期的趋势。

#### 仓位上限（Position Weight Cap）

**机制**：单只股票不超过总资产 25%。

**类比**：不把鸡蛋放在一个篮子里，每个篮子最多装 1/4。如果有一个篮子掉了，至少还有 3/4 的鸡蛋是安全的。

**为什么重要**：如果某只股票突然跌停，最大亏损就是 25%。如果没有仓位限制，万一全仓一只股票跌停，一天就亏 10%，小资金很难承受。

### 11.3 使用方法

```python
from big_a.strategy.real_trading import RealTradingStrategy
from big_a.backtest.engine import run_backtest_with_strategy

# 创建策略实例
strategy = RealTradingStrategy(
    signal=kronos_predictions,      # 预测信号
    topk=5,                         # 持仓 5 只股票
    n_drop=1,                       # 每周换 1 只
    risk_degree=0.95,               # 风险度（仓位）
    stop_loss=-0.08,                # 8% 止损
    max_weight=0.25,                # 单只股票最大权重 25%
    rebalance_freq=5,               # 每 5 个交易日调仓（每周）
)

# 运行回测
report, positions = run_backtest_with_strategy(strategy, config)
```

### 11.4 运行脚本

```bash
cd big-a && uv run python scripts/real_trading_backtest.py --help
```

常用参数：
- `--config`：回测配置文件
- `--output-dir`：输出目录
- `--kronos-config`：Kronos 模型配置
- `--skip-kronos-download`：跳过 Kronos 模型下载

### 11.5 与 TopkDropout 的对比

| 参数 | TopkDropout | RealTradingStrategy |
|------|-------------|---------------------|
| 持仓数量 | 50 只 | 5 只 |
| 调仓频率 | 每天（1 天） | 每周（5 天） |
| 每次换仓 | 5 只 | 1 只 |
| 止损机制 | 无 | 8% 止损 |
| 仓位上限 | 无 | 25% |
| 适用资金 | 1000 万以上 | 100 万左右 |
| 交易成本 | 高（每天换） | 低（每周换） |

**选择建议**：
- 大资金（1000万+）：用 TopkDropout，分散风险，收益更稳定
- 小资金（100万左右）：用 RealTradingStrategy，降低交易成本，严格风控
