# 数据管理模块详解

## 数据从哪里来？

### GitHub 数据仓库

Big-A 使用的数据来自 GitHub 上的 `chenditc/investment_data` 仓库。

你可能会想，为什么用 GitHub 上的数据，而不是自己从交易所获取？

直接从交易所获取数据有几个问题：
1. 需要付费的数据接口
2. 需要自己处理数据格式
3. 需要持续维护更新

而 `chenditc/investment_data` 仓库已经帮我们解决了这些问题：
1. 数据免费且开源
2. 已经转换成 Qlib 需要的格式
3. 定期更新

这就像你想吃披萨，你可以自己从种小麦开始做，也可以直接买已经做好的披萨底。显然后者更省心。

### 数据格式

Big-A 使用的数据是 Qlib 二进制格式，文件后缀是 `.bin`。

你可能会问，为什么不用 CSV 格式？CSV 不是更通用吗？

确实 CSV 更通用，但二进制格式有自己的优势：
1. **读取速度快**：二进制文件读取比解析 CSV 快很多。
2. **存储空间小**：压缩后的二进制文件比 CSV 小很多。
3. **支持高效查询**：Qlib 可以快速查询任意时间范围的数据。

想象一下，你要在一本厚厚的书里找某句话。如果书没有目录（像 CSV），你得一页页翻。如果书有详细的目录和索引（像二进制格式），你可以直接跳到对应页。

### 数据内容

数据包含 A 股所有股票的日频 OHLCV 数据。

OHLCV 是什么？这是五个英文单词的缩写：
- O = Open（开盘价）：每天第一笔成交价
- H = High（最高价）：当天最高成交价
- L = Low（最低价）：当天最低成交价
- C = Close（收盘价）：当天最后一笔成交价
- V = Volume（成交量）：当天成交的总股数

这五个数据是股票分析的基础，就像人的身高、体重、血压、心率、血型一样，是最基本的健康指标。

### 用图书馆类比

这个数据仓库就像一个已经整理好的图书馆。

- 每只股票是一个书架
- 每天是一本书
- 每本书里记录了当天的 OHLCV 数据

图书馆的管理员（chenditc/investment_data 项目）已经把所有书籍都编好了目录，分好了类。你只需要去借阅，不需要自己整理。

## 数据目录结构

### 完整目录树

Big-A 的数据存储在 `data/qlib_data/cn_data/` 目录下，结构如下：

```
data/qlib_data/cn_data/
├── calendars/
│   └── day.txt              # 交易日历（所有交易日列表）
├── instruments/
│   ├── all.txt             # 全部A股股票列表
│   ├── csi300.txt          # 沪深300成分股列表
│   └── csi500.txt          # 中证500成分股列表
└── features/               # 特征数据（二进制格式）
    ├── SH600000/           # 上证600000（浦发银行）
    │   ├── open.bday.bin   # 开盘价
    │   ├── high.bday.bin   # 最高价
    │   ├── low.bday.bin    # 最低价
    │   ├── close.bday.bin  # 收盘价
    │   └── volume.bday.bin # 成交量
    ├── SH600001/           # 上证600001
    │   └── ...
    └── SZ000001/           # 深证000001
        └── ...
```

### 各部分详解

#### calendars/day.txt

这个文件记录了所有的交易日。

为什么需要这个文件？因为 A 股不是每天都有交易。周末、节假日都不交易。如果没有交易日历，程序就不知道哪天是交易日，哪天不是。

这个文件的内容就像这样：
```
2010-01-04
2010-01-05
2010-01-06
2010-01-07
2010-01-08
2010-01-11
...
```

注意看，2010-01-09 和 2010-01-10 是周六和周日，所以不在列表里。

#### instruments/

这个目录包含各种股票池的列表。

- `all.txt`：所有 A 股股票
- `csi300.txt`：沪深300成分股（A股市场最有代表性的300只股票）
- `csi500.txt`：中证500成分股（A股市场中盘股的代表）

这些文件的内容就是股票代码列表，比如：
```
SH600000
SH600001
SZ000001
SZ000002
...
```

#### features/

这个目录是核心，存储所有股票的历史数据。

每只股票有一个单独的文件夹，文件夹名是股票代码。股票代码的格式是：
- SH + 6位数字 = 上海证券交易所
- SZ + 6位数字 = 深圳证券交易所

比如 `SH600000` 是上海证券交易所的浦发银行，`SZ000001` 是深圳证券交易所的平安银行。

每个股票文件夹里有多个 `.bday.bin` 文件，分别存储不同的数据字段。

## 数据更新流程

### 更新脚本位置

数据更新的主要代码在 `src/big_a/data/updater.py` 文件中。

这个脚本提供了几个关键函数，让我们一个一个来看。

### get_last_update_date()

这个函数的作用是读取 `calendars/day.txt` 文件的最后一行，返回最后一次更新的日期。

逻辑很简单：
1. 打开 `calendars/day.txt` 文件
2. 读取最后一行
3. 解析成日期格式
4. 返回这个日期

这就像查看冰箱上贴的购物清单最后一项，看看最后一次买菜是什么时候。

### update_incremental()

这个函数执行增量更新。它的流程是：

1. 调用 `get_last_update_date()` 获取最后更新日期
2. 从 GitHub 下载从那天到今天的数据包
3. 解压数据包到对应目录
4. 更新交易日历文件

为什么叫"增量更新"？因为只下载新增的数据，不是重新下载所有数据。这就像软件更新，只下载补丁，不重新下载整个软件。

### verify_update()

这个函数验证更新后的数据是否完整。

它会检查：
1. 文件是否存在
2. 文件大小是否合理
3. 数据格式是否正确

如果发现任何问题，会抛出异常，让更新过程失败。

这就像安装软件后验证安装是否成功，确保所有文件都正确放置了。

### 用"更新App"类比

整个数据更新流程就像手机 App 更新：

1. **检查版本** = `get_last_update_date()`：看看当前版本是多少
2. **下载更新** = `update_incremental()`：下载从当前版本到最新版本的更新包
3. **验证安装** = `verify_update()`：检查更新是否成功

整个过程是自动化的，你只需要运行一个命令。

## 数据验证

### 验证脚本位置

数据验证的代码在 `src/big_a/data/validation.py` 文件中。

数据验证非常重要，就像食品安全检测一样。如果数据有问题，再好的模型也得不到正确的结果。

### check_calendar_integrity()

这个函数检查交易日历是否连续。

逻辑是：
1. 读取交易日历的所有日期
2. 检查相邻两个交易日之间最多间隔3天（周五到周一）
3. 如果间隔超过3天，说明可能有数据缺失

比如：
- 周五到周一间隔3天，这是正常的
- 周一到周二间隔1天，这是正常的
- 周一到周三间隔2天，说明周二的数据缺失了

### check_price_continuity()

这个函数检测异常的价格变动。

A 股有涨跌停限制，正常情况下：
- ST 股票单日涨跌幅不能超过 5%
- 普通股票单日涨跌幅不能超过 10%
- 创业板和科创板单日涨跌幅不能超过 20%

如果发现某只股票单日涨跌超过限制，但又不是涨跌停，说明数据可能有错误。

比如某只股票从 10 元涨到 15 元，涨幅 50%，但又不是涨跌停，这肯定有问题。

### check_nan_ratio()

这个函数检查空值（NaN）的比例。

空值就是缺失的数据。如果某只股票某天的数据是空的，说明那天可能没有交易，或者数据采集出了问题。

函数会计算每只股票、每个字段的空值比例。如果比例过高，会发出警告。

这就像体检报告，如果某项指标异常高，医生会特别关注。

### check_stock_coverage()

这个函数检查股票池的完整性。

比如检查沪深300成分股，应该有大约 300 只股票。如果只有 200 只，说明有 100 只股票的数据缺失。

不同的股票池有不同的要求：
- 沪深300应该有约300只股票
- 中证500应该有约500只股票
- 全部A股应该有几千只股票

### generate_data_report()

这个函数生成完整的数据质量报告。

它会调用上面所有的检查函数，然后生成一个详细的报告，包括：
1. 总体统计（股票数量、日期范围、数据完整性）
2. 交易日历检查结果
3. 价格连续性检查结果
4. 空值比例检查结果
5. 股票池覆盖检查结果
6. 任何发现的问题和建议

这就像体检报告，把所有检查结果汇总在一起，给出总体评价。

## 常用命令

### 查看数据状态

```bash
uv run python scripts/update_data.py --dry-run
```

这个命令不会实际下载数据，只是检查当前数据的状态。

它会显示：
- 最后更新日期
- 当前数据覆盖的日期范围
- 股票数量
- 是否需要更新

`--dry-run` 意思是"空运行"或"试运行"，就像在电脑上模拟飞行，不会真的起飞。

### 下载数据

```bash
uv run python scripts/update_data.py update
```

这个命令会执行实际的数据更新。

如果数据已经很新，可能会提示"无需更新"。如果需要更新，会自动下载并安装。

这个过程可能需要几分钟时间，取决于数据量大小和网络速度。

### 验证数据

```bash
uv run python scripts/validate_data.py
```

这个命令会对当前数据进行全面验证。

它会运行所有的检查函数，然后生成一个详细的报告。

如果发现任何问题，会在报告中明确指出。如果一切正常，会显示"数据验证通过"。

建议定期运行这个命令，比如每月一次，确保数据质量。

## 数据管理的重要性

你可能会想，数据管理看起来很枯燥，为什么要花这么多篇幅讲它？

其实数据管理是量化研究的基础，就像盖房子需要打好地基。

如果地基不稳，房子再漂亮也会出问题。同样，如果数据有问题，模型再先进也得不到好结果。

在量化研究中，有一个说法："垃圾进，垃圾出"。意思就是，如果输入的数据质量差，输出的结果也不会好。

所以，花时间了解和管理数据，是非常值得的投资。

## 常见问题

### 数据多久更新一次？

这个数据仓库通常每周更新一次。你可以运行 `--dry-run` 命令查看最后更新日期，判断是否需要更新。

### 数据可以离线使用吗？

可以。数据下载到本地后，所有操作都可以离线进行。只有更新数据时才需要联网。

### 数据有多久的历史？

这个数据仓库从 2010 年开始，涵盖了十多年的历史数据。对于量化研究来说，这个时间长度足够了。

### 可以只用部分股票吗？

可以。你可以通过修改配置文件，只使用沪深300、中证500或者自己定义的股票池。

### 数据文件很大怎么办？

数据文件确实很大，可能有几十 GB。建议：
1. 使用 SSD 硬盘，提高读取速度
2. 定期清理不需要的数据
3. 使用压缩工具减少存储空间
