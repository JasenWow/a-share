# 实盘交易策略（RealTradingStrategy）

## 1. 什么是 RealTradingStrategy？

RealTradingStrategy 是 Big-A 为小资金实盘交易设计的策略。它继承自 TopkDropoutStrategy，在其基础上增加了三大风控机制：止损、周频调仓、仓位上限。

如果说 TopkDropout 是一个"激进型"选手，那 RealTradingStrategy 就是一个"保守型"选手。它牺牲了一部分潜在收益，换来更低的风险和更少的交易成本。

**设计背景：**

小资金（约100万）和大资金（约1亿）的玩法完全不同：
- 大资金持有50只股票，每只约2万元，交易成本可以忽略
- 小资金持有50只股票，每只约2万元，佣金最低5元就占了0.025%

小资金需要更少的持仓、更低的换手、更严格的风控。RealTradingStrategy 就是为这个场景设计的。

## 2. 三大风控机制详解

### 2.1 止损机制（Stop Loss）

**机制：** 每天检查所有持仓股票，如果单只股票亏损超过 8%，立即卖出。

**代码逻辑：**

```python
# 计算盈亏
pnl = (current_price - entry_price) / entry_price

# 如果亏损超过 8%，触发止损
if pnl <= self.stop_loss:  # stop_loss = -0.08
    # 立即卖出全部持仓
    sell_order = Order(stock_id=code, amount=amount, direction=Order.SELL)
```

**类比：** 止损就像家里的保险丝。当电流超过安全值时，保险丝会自己烧断，防止电线着火。止损就是在亏损超过 8% 时自动"烧断"，防止亏损继续扩大。

**为什么是 8%？**

8% 是一个经验值：
- 太小（比如 3%）：容易触发，错过反弹
- 太大（比如 15%）：亏损太大，小资金承受不起

8% 意味着单只股票最多亏总资金的 2%（假设持仓上限 25%）。即使连续止损两次，也只亏 4%，还在可控范围内。

**实际例子：**

假设你以 10 元买入某股票：
- 跌到 9.2 元（亏损 8%）：触发止损，立即卖出
- 即使后面反弹到 8 元，你也已经在 9.2 元逃跑了
- 如果没有止损，可能一直持有到 8 元，亏 20%

### 2.2 周频调仓（Weekly Rebalancing）

**机制：** 每 5 个交易日才执行一次选股和调仓，不是每天调整。

**代码逻辑：**

```python
self._step_count += 1
if self._step_count % self.rebalance_freq == 0:
    # 只有在第 5、10、15... 个交易日才执行调仓
    rebalance_decision = super().generate_trade_decision(execute_result)
```

**类比：** 周频调仓就像每周只去一次超市买菜，而不是每天去。
- 每天去：既累又费油，还容易冲动消费
- 每周去：省力又省钱，还能规划好一周的菜谱

**为什么周频而不是日频？**

A 股交易成本不低：
- 买入佣金：0.025% + 过户费 0.001% = 0.026%
- 卖出佣金：0.025% + 印花税 0.1% + 过户费 0.001% = 0.126%

每次买卖的摩擦成本约 0.15%。如果每天换 10% 的仓位：
- 日频：每天 0.15% × 10% = 0.015% 的成本
- 周频：每周 0.15% × 10% = 0.015%，但一年只有 50 次，而不是 250 次

周频调仓可以把年交易成本降低 80%。

### 2.3 仓位上限（Position Weight Cap）

**机制：** 任何单只股票的权重不超过 25%。

**代码逻辑：**

```python
for order in buy_orders:
    existing_weight = existing_value / total_value
    new_weight = buy_value / total_value

    # 如果超过 25% 上限，按比例缩减
    if existing_weight + new_weight > self.max_weight:  # 0.25
        max_value = self.max_weight * total_value
        allowed_buy_value = max_value - existing_value
        allowed_amount = allowed_buy_value / buy_price
        order.amount = round_amount_by_trade_unit(allowed_amount, factor)
```

**类比：** 仓位上限就像规定"每个篮子里最多放 1/4 的鸡蛋"。即使某个篮子再好，也不能放太多，防止这个篮子掉了全部打碎。

**为什么是 25%？**

25% 意味着即使某只股票跌停（-10%），总资产也只亏 2.5%（25% × 10%）。这个损失是小资金可以承受的。

**实际例子：**

假设总资产 100 万：
- 仓位上限 25%：某只股票最多买 25 万
- 该股跌停（-10%）：亏损 2.5 万，剩余 97.5 万
- 如果没有仓位上限，全仓买入了该股：亏损 10 万，剩余 90 万

## 3. 与 TopkDropout 的对比

### 3.1 核心差异

| 特性 | TopkDropout | RealTradingStrategy |
|------|-------------|---------------------|
| 持仓数量 | 50 只 | 5 只 |
| 调仓频率 | 每天 | 每 5 天（周频） |
| 止损 | 无 | 8% 自动止损 |
| 仓位上限 | 无 | 25% 上限 |
| 交易成本 | 较高 | 较低 |
| 风险等级 | 较高 | 较低 |
| 适用资金 | 1000 万以上 | 100 万左右 |

### 3.2 选股逻辑对比

**TopkDropout：**
- 每天从 3000 只股票中选分数最高的 50 只
- 每天淘汰表现最差的 5 只
- 表现差的股票最多持有 1 天

**RealTradingStrategy：**
- 每 5 天从 3000 只股票中选分数最高的 5 只
- 每周淘汰表现最差的 1 只
- 止损触发时立即卖出（不等调仓日）

### 3.3 什么时候用哪个？

**用 TopkDropout：**
- 回测研究阶段，想看看模型的真实选股能力
- 大资金（1000 万以上），需要充分分散风险
- 追求更高收益，可以承受更大波动

**用 RealTradingStrategy：**
- 准备实盘交易，需要模拟真实的风控限制
- 小资金（100 万左右），交易成本敏感
- 风险厌恶型，不想承受大幅回撤
- 想要一个"省心"的策略，不想每天盯盘

## 4. 配置参数详解

### 4.1 完整配置示例

配置文件：`configs/backtest/real_trading.yaml`

```yaml
strategy:
  class: RealTradingStrategy
  module_path: big_a.strategy.real_trading
  kwargs:
    signal: <PRED>              # 预测信号（由 Kronos 或 LightGBM 生成）
    topk: 5                    # 持仓 5 只股票
    n_drop: 1                  # 每周换 1 只
    risk_degree: 0.95           # 风险度（仓位），0.95 表示用 95% 的资金

risk_controls:
  stop_loss: -0.08            # 止损阈值：亏损超过 8% 卖出
  max_weight: 0.25            # 仓位上限：单只股票最多 25%
  rebalance_freq: 5            # 调仓频率：每 5 个交易日调仓一次

backtest:
  start_time: "2022-01-01"    # 回测开始日期
  end_time: "2024-12-31"      # 回测结束日期
  account: 1000000            # 初始资金：100 万
  benchmark: SH000300         # 基准：沪深 300
  exchange_kwargs:
    freq: day                 # 频率：日线
    limit_threshold: 0.095     # 涨跌停阈值：9.5%
    deal_price: close          # 成交价：收盘价
    open_cost: 0.0005          # 买入成本：0.05%
    close_cost: 0.0015         # 卖出成本：0.15%
    min_cost: 5                # 最低佣金：5 元
```

### 4.2 参数说明

**策略参数（strategy.kwargs）：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| signal | 必填 | 预测信号，DataFrame，包含 datetime 和 instrument 索引 |
| topk | 5 | 持仓股票数量 |
| n_drop | 1 | 每次调仓淘汰的数量 |
| risk_degree | 0.95 | 风险度，0.95 表示使用 95% 的资金 |

**风控参数（risk_controls）：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| stop_loss | -0.08 | 止损阈值，-0.08 表示亏损 8% 触发止损 |
| max_weight | 0.25 | 单只股票最大权重，0.25 表示 25% |
| rebalance_freq | 5 | 调仓频率，5 表示每 5 个交易日调仓一次 |

### 4.3 参数调整建议

**更激进（追求更高收益）：**
- 增加 topk：5 → 8
- 增加 n_drop：1 → 2
- 放宽止损：-0.08 → -0.10
- 放宽仓位上限：0.25 → 0.30

**更保守（追求更低风险）：**
- 减少 topk：5 → 3
- 减少 n_drop：1 → 1（不变）
- 收紧止损：-0.08 → -0.06
- 收紧仓位上限：0.25 → 0.20
- 增加 rebalance_freq：5 → 10（两周一次）

## 6. 回测结果解读

### 6.1 关键指标

运行完回测后，会输出类似以下的摘要：

```
===== 回测摘要 =====
回测期间: 2022-01-01 至 2024-12-31
初始资金: 1,000,000 元
最终资金: 1,234,567 元
总收益率: 23.46%
年化收益率: 10.89%
最大回撤: -8.32%
夏普比率: 1.23
信息比率: 0.67
止损次数: 45
平均持仓: 4.2 只
调仓次数: 120
==================
```

### 6.2 各指标含义

**总收益率：** 策略在整个回测期间的总收益。23.46% 意味着 100 万变成了 123.46 万。

**年化收益率：** 把总收益折算成每年的收益。10.89% 比存款和理财高，但也不算特别激进。

**最大回撤：** 从最高点到最低点的跌幅。-8.32% 意味着策略曾经从高点跌了 8.32%，但最后还是涨回来了。

**夏普比率：** 每承担一单位风险，能获得多少超额收益。1.23 属于良好水平（>1 就不错了）。

**信息比率：** 策略相对于基准（沪深300）的超额收益。0.67 表示策略每年跑赢基准 0.67%。

**止损次数：** 整个回测期间触发了多少次止损。45 次平均每年 15 次，属于正常范围。

**平均持仓：** 平均持有多少只股票。4.2 只说明大部分时间持仓在 4-5 只之间。

**调仓次数：** 总共调仓了多少次。120 次（3 年约 750 个交易日 / 5 天 ≈ 150 次，实际 120 次因为有节假日）。

### 6.3 怎么看净值曲线

好的净值曲线应该：
- 整体向上（赚钱）
- 波动较小（风险低）
- 下跌后能涨回来（韧性足）

RealTradingStrategy 的净值曲线通常比 TopkDropout 更平滑，因为：
- 持仓更集中（5 只 vs 50 只）
- 调仓更少（每周 vs 每天）
- 有止损保护（防止大跌）

## 7. 实盘使用建议

### 7.1 回测到实盘的过渡

**第一步：确认回测结果**
- 年化收益 > 0（废话，但很重要）
- 最大回撤 < 20%（可接受）
- 夏普比率 > 0.5（有一定预测能力）

**第二步：小资金验证**
- 用 10 万（实盘的 10%）跑一段时间
- 观察策略表现和回测是否一致
- 记录每次调仓的理由

**第三步：逐步放大**
- 确认策略有效后，逐步放大到 50 万
- 再放大到 100 万
- 不要一下子全部投入

### 7.2 日常操作

**每天开盘前：**
- 检查是否有股票触发止损（系统自动卖出）
- 看看持仓股票的消息面有没有大新闻

**每周一：**
- 查看上周的调仓报告
- 如果发现异常（比如连续多只股票止损），考虑暂停策略

**每月：**
- 回顾本月的策略表现
- 和回测对比，看是否有大幅偏离
- 必要时调整参数

### 7.3 注意事项

**1. 止损的局限性：**

如果股票连续跌停（比如突发利空），可能无法在 -8% 时卖出。

例子：
- 某股因重组失败，停牌后复牌直接跌停（-10%）
- 止损价是 9.2 元，但开盘价是 8.5 元
- 实际卖在 8.5 元，亏 15%

这是止损的盲区，无法完全避免。分散持仓（单只不超过 25%）可以缓解这个问题。

**2. 滑点的影响：**

回测假设按收盘价成交，实盘可能有一定滑点。

滑点 0.2% 听起来不多，但：
- 每周换 1 只股票（1/5 = 20% 换手率）
- 每年 50 周，换手率 1000%
- 滑点损失：1000% × 0.2% = 2%

这个损失不算大，但也要有心理准备。

**3. 模型衰减：**

量化模型的预测能力会随时间衰减。

原因：
- 市场风格变化（以前有效的因子可能失效）
- 参与者增加（超额收益被稀释）
- 交易成本上升

建议每 3-6 个月重新训练模型，或至少检查模型的 IC 是否还为正。

## 8. 常见问题

**Q：止损会不会卖得太早，错过反弹？**

A：会的。但这是风控的代价。与其等待反弹（可能等来更大的下跌），不如先保住本金。8% 的止损线是一个平衡点。

**Q：为什么只有 5 只持仓，不能多一点吗？**

A：可以调整 topk 参数。但小资金如果持仓太多，每只股票只买几万元，佣金最低 5 元就占了 0.05%，加上滑点，成本会很高。

**Q：周频调仓会不会错过机会？**

A：会。但机会和风险是并存的。周频调仓让你不会追涨杀跌，也不会被短期波动洗出去。

**Q：如果止损和调仓同一天触发，先执行哪个？**

A：先执行止损。当天触发止损的股票会立即卖出，不会参与当天的调仓。

**Q：持仓股票停牌了怎么办？**

A：停牌期间无法买卖，只能等复牌。止损机制在停牌期间不工作，复牌后如果直接跌停，可能亏得比较多。

**Q：能不能同时用 RealTradingStrategy 和 TopkDropout？**

A：不建议。两套逻辑冲突：
- RealTradingStrategy 是周频调仓
- TopkDropout 是日频调仓

混用会导致持仓混乱。

## 9. 总结

RealTradingStrategy 是一个为小资金实盘设计的保守型策略。它的核心理念是：

**保住本金，稳健增值。**

三大风控机制：
1. **止损**：亏损 8% 自动卖出，防止深套
2. **周频调仓**：每周换一次，减少交易成本
3. **仓位上限**：单只股票最多 25%，防止黑天鹅

**适用场景：**
- 小资金（100 万左右）
- 风险厌恶型投资者
- 不想每天盯盘

**不适用场景：**
- 大资金（1000 万以上）
- 追求高收益、能承受高波动
- 需要日频调仓的策略

记住：
- 回测好不代表实盘好
- 实盘前务必先用小资金验证
- 持续监控策略表现，及时调整参数
