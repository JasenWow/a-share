# 模型模块详解

## 1. 模型是什么？

想象你在看天气预报。气象台要预测明天的天气，需要分析今天的温度、湿度、风向、气压等各种数据，然后根据历史规律给出预测。

在量化交易中，模型就是"天气预报员"。它分析股票的历史价格、成交量、技术指标等各种数据，学习历史规律，然后预测未来这只股票是涨还是跌。

只不过，气象台的预报员是经验丰富的人类专家，而我们的"预报员"是数学模型，通过学习成千上万的历史案例来做出判断。

## 2. 为什么需要两种模型？

Big-A 项目使用了两种完全不同的预测模型：LightGBM 和 Kronos。

### LightGBM：经验丰富的老股民

就像一位在股市摸爬滚打多年的老股民，LightGBM 基于传统的统计学方法。它擅长发现明显的规律，比如：

- "连续大涨三天后，第四天下跌的概率是 60%"
- "成交量突然放大，通常意味着变盘"
- "市盈率低于 10 的股票，长期表现不错"

这些规律虽然简单，但在大多数时候都很有效。

### Kronos：学习了大量案例的 AI

就像一位读过百万份财报、看过亿万条 K 线图的超级 AI，Kronos 基于深度学习。它能发现人类难以察觉的复杂模式，比如：

- 某个技术指标的微妙变化，结合成交量形态，预示着未来的走势
- 多只股票之间的关联性如何影响单只股票的表现
- 不同市场环境下的资金流向规律

这些模式可能非常复杂，但深度学习模型能够捕捉到。

**为什么两者都要？**

就像投资不能只听一个人的意见一样，两种方法各有优势：

- LightGBM 快速、稳定、容易理解，适合作为基础判断
- Kronos 能发现复杂模式，提供不同的视角

当两个模型的意见一致时，信心更足。当意见不同时，可以进一步研究原因。

## 3. LightGBM 详解

### 3.1 什么是决策树？

决策树就像玩"20 问"游戏。

假设你要猜一只股票明天是涨还是跌，你可以问一系列问题：

1. "今天涨了吗？" → 是 → 继续问
2. "成交量比昨天大吗？" → 否 → 继续问
3. "市盈率小于 20 吗？" → 是 → 预测：明天涨

每个问题就是一个"节点"，每个答案就是一个"分支"，最后的结论就是"叶子"。

一棵决策树就是一套清晰的判断规则。但单棵决策树太简单了，就像只问三五个问题就下结论，容易出错。

### 3.2 什么是梯度提升？

梯度提升就像组建一个专家团队。

第一步：找第一个"专家"（决策树），让他做出预测。肯定会有错误。

第二步：分析第一个专家的错误，找第二个"专家"专门纠正这些错误。

第三步：分析前两个专家的联合错误，找第三个专家继续纠正。

重复这个步骤，比如 100 次，就得到了 100 个"专家"。最后把这 100 个专家的预测结果加权平均，就得到了更准确的预测。

这就是"梯度提升"的核心思想：把很多个"弱学习器"（每棵树都不完美）组合成一个"强学习器"（整体预测很准确）。

### 3.3 训练过程详解

**第一步：准备数据**

就像给学生准备教材，我们需要：

- **特征**（输入数据）：Alpha158（158 个技术指标）+ 9 个自定义因子 = 167 个特征
  - 比如：5 日均线、20 日均线、RSI、MACD、成交额变化等
- **标签**（目标数据）：未来 1 天的收益率
  - 公式：`Ref($close, -2) / Ref($close, -1) - 1`
  - 翻译：用后天的收盘价除以明天的收盘价，再减 1
  - 为什么要用后天而不是明天？因为 A 股 T+1 交易制度，今天买的股票明天才能卖

**第二步：训练模型**

把历史数据喂给模型，模型就像学生一样，通过反复练习（迭代）来学习规律。

每次迭代，模型都会：
1. 用当前的规则预测
2. 计算预测误差（MSE，均方误差）
3. 调整规则，减少误差

**第三步：验证模型**

用一部分没参与训练的数据测试模型，看看它的预测准确率如何。就像考试一样，验证学习成果。

### 3.4 关键超参数解释

超参数就像是模型的"设定值"，需要我们手动调整。

| 参数 | 数值 | 类比 | 作用 |
|------|------|------|------|
| max_depth | 8 | 问题的深度 | 最多问 8 个问题就下结论，避免问太多导致过拟合 |
| num_leaves | 210 | 答案的数量 | 决策树最多有 210 种不同的结论 |
| learning_rate | 0.2 | 学习速度 | 每次迭代调整幅度的比例，越大学习越快但不稳定 |
| colsample_bytree | 0.8879 | 随机抽样比例 | 每棵树只随机选 88.79% 的特征，避免过度依赖某些特征 |
| subsample | 0.8789 | 样本抽样比例 | 每次迭代只用 87.89% 的数据，增加模型多样性 |

**为什么需要这些参数？**

就像老师教书一样：
- max_depth 太小，学生学不到深入的知识（欠拟合）
- max_depth 太大，学生会死记硬背（过拟合）
- learning_rate 太大，学生学得快但不扎实
- learning_rate 太小，学生学得太慢

需要在速度和准确性之间找到平衡。

## 4. Kronos 详解

### 4.1 什么是 Transformer？

Transformer 就像一位能理解上下文的阅读高手。

假设你读这句话："银行账户里的钱很多"。

- 如果前文是"抢劫案"，"银行"指建筑物
- 如果前文是"理财"，"银行"指金融机构

人类能根据上下文理解词义，Transformer 模型也能。

在股票预测中，Transformer 能理解：
- 今天的大涨是在什么背景下发生的（牛市、熊市、震荡市？）
- 某只股票的走势与整体市场的关系
- 不同技术指标之间的隐含联系

Transformer 通过"注意力机制"（Attention）来实现这一点，就像人类读文章时会重点关注某些关键词一样。

### 4.2 什么是 Tokenizer？

Tokenizer 就像把连续的信息离散化。

想象你在写一篇文章：
- 文章是连续的文字流
- 但计算机处理时，把它拆成一个个词（token）
- 比如："今天天气真好" → ["今天", "天气", "真", "好"]

Kronos 的 Tokenizer（BSQ，二进制球面量化器）把连续的股价数据变成离散的 token：

- 输入：连续的 OHLCV 数据（开盘价、最高价、最低价、收盘价、成交量）
- 处理：二进制量化，把每个价格变成一串 0 和 1
- 输出：一串 token，比如 `[101, 011, 110, ...]`

为什么要这样做？

- 股价是连续的，但模型处理离散数据更高效
- 量化后的数据抗噪能力强，小幅波动不影响 token
- 类似人类把"今天涨了 0.01%"和"涨了 0.02%"都理解为"微涨"

### 4.3 什么是自回归预测？

自回归预测就像写作文，一个字一个字地写。

传统方法：一次预测未来 10 天的全部价格 → 容易出错，因为后面的预测依赖前面的预测

自回归方法：
1. 预测明天 → 得到 token A
2. 把 A 加入历史 → 预测后天 → 得到 token B
3. 把 B 加入历史 → 预测大后天 → 得到 token C
4. ... 重复 10 次

每一步预测都基于之前所有的预测结果，就像写文章时，写完"今天"会继续写"天气"，然后"真"，然后"好"，每一步都考虑前面写了什么。

### 4.4 信号生成流程

Kronos 生成交易信号的完整流程：

**第一步：准备历史数据**

取每只股票过去 90 个交易日的 OHLCV 数据（开盘价、最高价、最低价、收盘价、成交量）

- 为什么是 90 天？大约 4 个多月，包含足够的短期和中期信息
- 数据形状：`(90, 5)`，90 行 5 列

**第二步：标准化**

用 z-score 标准化：`(x - mean) / std`

就像把不同班级的考试成绩标准化一样：
- 班级 A 平均分 80，标准差 10
- 班级 B 平均分 60，标准差 5
- 同样的 85 分，在 A 班是中等偏上，在 B 班是顶尖

标准化后的数据：
- 均值为 0
- 标准差为 1
- 不同股票、不同时期的数据可以比较

**第三步：Tokenize**

把标准化后的 OHLCV 数据通过 BSQ tokenizer 变成 token 序列。

**第四步：自回归预测**

用 Transformer 模型预测未来 10 个交易日的 token，然后：

1. 把 token 反量化成价格
2. 反标准化（乘以标准差，加上均值）
3. 得到预测的收盘价序列

**第五步：计算信号**

```
信号 = 未来 10 天预测收盘价的均值 - 最后一天的实际收盘价
```

如果信号 > 0，预测上涨
如果信号 < 0，预测下跌

**为什么用均值？**

避免单日波动的影响，看整体趋势。就像评估一个人，不能只看一天的表现，要看长期的平均水平。

### 4.5 为什么用预训练模型？

预训练模型就像"站在巨人的肩膀上"。

**从头训练的问题：**

- 需要海量的数据（数百万条 K 线）
- 需要强大的计算资源（多个 GPU 训练数周）
- 需要深厚的深度学习知识
- 时间成本极高

**使用预训练模型的优势：**

- Kronos 已经在海量的全球股票数据上训练好了
- 学习到了通用的价格模式
- 我们只需要"微调"或直接使用
- 省时省力，效果有保证

就像：
- 从头造汽车 → 需要设计、制造、测试，数年
- 买现成的车 → 直接开，几分钟

Kronos 来自 HuggingFace 的 `NeoQuasar/Kronos-base`，已经在全球股票数据上训练过，可以直接用于 A 股。

## 5. 两种模型的对比

| 维度 | LightGBM | Kronos |
|------|----------|--------|
| **原理** | 决策树 + 梯度提升 | Transformer + 自回归预测 |
| **类比** | 经验丰富的老股民 | 学习了大量案例的 AI |
| **训练数据** | Alpha158 + 9 因子（167 个特征） | 原始 OHLCV 数据（5 个特征） |
| **训练方式** | 从头训练 | 使用预训练权重 |
| **预测速度** | 非常快（毫秒级） | 较慢（秒级） |
| **可解释性** | 强（可以看到决策路径） | 弱（黑盒模型） |
| **数据需求** | 较少（几千条样本即可） | 很多（预训练需要海量数据） |
| **适用场景** | 寻找明显的统计规律 | 发现复杂的非线性模式 |
| **计算资源** | CPU 即可 | 推荐 GPU/MPS |
| **维护成本** | 低 | 中 |
| **改进空间** | 调参、增加特征 | 微调、换更大的模型 |
| **擅长** | 短期趋势、信号稳定 | 长期模式、复杂关系 |

## 6. 如何选择模型？

没有绝对的好坏，看场景：

**选择 LightGBM，如果：**

- 追求预测速度（比如每天需要实时预测 5000 只股票）
- 需要可解释性（比如向客户解释为什么推荐这只股票）
- 数据量不大（只有几年的历史数据）
- 计算资源有限（没有 GPU）
- 偏好简单、稳定的方案

**选择 Kronos，如果：**

- 有充足的计算资源（Mac 的 MPS 或 NVIDIA 的 GPU）
- 想尝试深度学习方法
- LightGBM 的表现遇到瓶颈
- 对可解释性要求不高
- 想探索更复杂的模式

**两者都用，如果：**

- 有足够的资源和时间
- 想通过模型融合提高准确性
- 作为研究项目，想对比不同方法

## 7. 常用命令示例

### 7.1 训练 LightGBM 模型

```bash
# 使用默认配置训练
python -m big_a.scripts.train_model --model lightgbm

# 使用自定义配置训练
python -m big_a.scripts.train_model \
    --model lightgbm \
    --config configs/model/lightgbm.yaml \
    --dataset qlib_data

# 训练并验证
python -m big_a.scripts.train_model \
    --model lightgbm \
    --validate \
    --test
```

### 7.2 使用 Kronos 模型预测

```bash
# 预测单只股票
python -m big_a.scripts.predict \
    --model kronos \
    --instrument 000001.SZ \
    --start_date 2024-01-01 \
    --end_date 2024-12-31

# 预测多只股票
python -m big_a.scripts.predict \
    --model kronos \
    --instruments 000001.SZ,000002.SZ,600000.SH \
    --output predictions.csv

# 批量预测（从文件读取股票列表）
python -m big_a.scripts.predict \
    --model kronos \
    --instrument_list stock_list.txt \
    --output predictions.pkl
```

## 8. 总结

模型是量化交易的核心，就像天气预报员的预测能力决定了天气预报的准确性。

- **LightGBM**：快速、稳定、易理解，适合大多数场景
- **Kronos**：强大、复杂、潜力大，适合探索深度学习

对于初学者，建议从 LightGBM 开始，理解基本原理后再尝试 Kronos。两种方法各有价值，关键是要根据实际需求选择。

记住，模型只是工具，真正决定投资收益的是：
- 清晰的投资逻辑
- 严格的风险控制
- 持续的优化迭代

模型能帮我们更好地判断，但不能保证赚钱。投资有风险，入市需谨慎。
