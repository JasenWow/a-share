# 项目架构总览

## 项目定位

### Big-A 是什么

Big-A 是一个基于 Qlib 的 A 股量化交易研究框架。

你可能会问，这到底是什么意思？让我用最简单的话解释一下。

量化交易就是用数学和统计学的方法来分析股票数据，然后让计算机帮我们做投资决策。就像医生通过各种检查指标来诊断病人一样，量化交易通过分析股票的各种数据来判断哪些股票更有可能赚钱。

Qlib 是微软开源的一个量化投资研究平台，提供了强大的数据处理、因子计算和回测功能。Big-A 就是在 Qlib 的基础上，专门针对 A 股市场做了一整套完整的研究工具。

### 核心目标

Big-A 的核心目标很明确，让量化研究从"手工操作"变成"自动化流水线"。

想象一下，如果没有这个框架，你需要手动下载数据、手动写代码计算各种指标、手动训练模型、手动回测策略。这就像用手工锤子砸钉子盖房子一样，效率低、容易出错。

有了 Big-A，就像有了一套电动工具，每一步都有现成的工具可以用。数据下载有一键脚本，因子计算有现成的函数，模型训练有统一的接口，回测评估有完整的报告生成系统。

### 用工厂类比

如果把这个项目比作一个工厂，那它的运作流程是这样的：

- **原材料（数据）**：股票的成交数据，就像工厂的原料
- **加工（因子计算）**：从原始数据中计算出各种有用的特征，就像把原料加工成半成品
- **生产（模型训练）**：用这些特征训练出能预测股票涨跌的模型，就像把半成品组装成成品
- **质检（回测评估）**：用历史数据测试模型表现，就像检查成品质量

## 整体数据流

### 文字版流程图

整个项目的数据流动过程就像一条流水线：

```
数据下载 → 数据验证 → Qlib初始化 → 特征工程(Alpha158) → 模型训练(LightGBM/Kronos)
→ 预测打分 → 策略选股(TopkDropout) → 回测模拟 → 评估分析 → 可视化报告
```

每个环节都有明确的目标和产出。

### 用"做菜"类比

让我们用做菜的过程来理解这个流程：

1. **数据下载** = 买菜。你去菜市场把所有需要的食材都买回来，这是后续所有步骤的基础。

2. **数据验证** = 检查食材。看看菜是不是新鲜的，有没有坏的。如果数据有问题，后面的步骤就没意义了。

3. **Qlib初始化** = 准备厨房。把锅碗瓢盆摆好，各种调料准备就绪，确保烹饪环境就绪。

4. **特征工程(Alpha158)** = 洗菜切菜。把原始的股价数据加工成模型能理解的特征。比如计算均线、波动率、动量等，就像把菜洗干净切好。

5. **模型训练** = 炒菜。用历史数据训练模型，让模型学会从特征中找出规律，这就像厨师掌握火候和调料的搭配。

6. **预测打分** = 判断菜的味道。模型对新的股票数据进行预测，给出每只股票的得分，就像厨师判断这道菜的味道如何。

7. **策略选股** = 挑选最好的菜。根据模型得分，用 TopkDropout 策略选出表现最好的股票，就像从所有菜里挑出最好吃的几道。

8. **回测模拟** = 试吃。用历史数据模拟整个投资过程，看看这个策略在过去的表现如何，就像试吃菜品看味道好不好。

9. **评估分析** = 评价。计算各种指标，看看这个策略的收益率、风险、回撤等表现如何，就像给菜品打分。

10. **可视化报告** = 摆盘上桌。把所有的分析结果做成漂亮的图表和报告，就像把菜摆好端上桌。

## 模块功能概览

### config：配置管理

负责加载和合并配置文件。

想象你有一台咖啡机，上面有很多按钮和旋钮。你不需要每次都重新调，只要保存一个配置，下次一键调用就行。config 模块就是管理这些"设置"的地方。

### qlib_config：Qlib 初始化

初始化 Qlib 数据引擎，注册自定义的操作符。

就像给新电脑安装操作系统和驱动程序，确保后续的所有功能都能正常运行。

### data：数据管理

负责下载和验证 A 股数据。

这就像是图书馆的图书采购部门，负责从供应商那里获取书籍，然后检查书籍是否完整、是否有损坏。

### factors：因子计算

计算选股因子，也就是模型需要的特征。

就像体检时医生会检查身高、体重、血压、血糖等各种指标。factors 模块就是计算股票的各种"体检指标"。

### models：模型管理

训练模型、加载模型、生成预测。

这就像是训练一个新手医生。给他看很多病例（训练数据），让他学习怎么诊断疾病。训练好了之后，他就能给新病人看病（预测）了。

### backtest：回测引擎

模拟交易过程，计算收益。

就像在电脑上玩炒股模拟游戏。用历史数据模拟真实的买卖过程，看看如果按照某个策略操作，能赚多少钱。

### evaluation：评估分析

评估模型和策略的表现。

就像考试后分析成绩。不仅看总分，还要看哪些题做错了、哪些知识点没掌握，这样才能不断改进。

### experiment：实验管理

记录实验参数和结果。

就像实验室的实验记录本。每次做实验都记下用了什么参数、得到了什么结果，方便以后对比和分析。

### scripts：命令行工具

提供各种命令行入口脚本。

就像手机上的各种 App 图标。你可以直接点开使用，不需要知道背后的复杂代码。

## 配置系统详解

### 为什么用配置文件

你可能会问，为什么要把这些设置写在配置文件里，直接写在代码里不行吗？

确实可以，但用配置文件有很多好处：

1. **易于修改**：想改个参数，打开文件改一下就行，不需要改代码。
2. **易于管理**：不同实验用不同配置，方便对比。
3. **易于分享**：把配置文件分享给别人，别人就能复现你的实验。
4. **易于版本控制**：Git 可以追踪配置文件的变化历史。

这就像做菜。你可以每次都临时想放多少盐，也可以按照菜谱来。配置文件就是量化研究的"菜谱"。

### YAML 配置的层级结构

Big-A 使用 YAML 格式的配置文件。YAML 是一种人类友好的配置文件格式，读起来很直观。

举个例子：

```yaml
qlib_init:
  provider_uri: data/qlib_data/cn_data
  region: cn

model:
  name: lightgbm
  class: LGBModel
  module_path: qlib.contrib.model.gbdt

dataset:
  class: Alpha158
  module_path: qlib.data.dataset
```

这种层级结构就像文件夹的层级。qlib_init 里面有自己的设置，model 里面有自己的设置，dataset 里面也有自己的设置。

### deep-merge 是什么

这是 Big-A 配置系统的一个重要特性。

想象你有一个基础配置文件 base.yaml，里面写了一些通用的设置。然后你有一个 lightgbm.yaml，里面写了一些针对 LightGBM 模型的特定设置。

deep-merge 就是把这两个文件合并在一起。如果两个文件都有同一个设置，就用 lightgbm.yaml 里的值覆盖 base.yaml 里的值。如果只有 base.yaml 里有某个设置，就保留它。

这就像装修房子。你有一个基础设计方案，然后针对客厅、卧室、厨房分别有特定的调整方案。最终的效果就是基础方案加上各个房间的特定调整。

### 实际例子

Big-A 中配置加载是这样的：

```python
from big_a.config import load_config

config = load_config(
    "configs/base.yaml",
    "configs/model/lightgbm.yaml",
    "configs/data/handler_alpha158.yaml"
)
```

这三个文件会按照 deep-merge 的规则合并：

- base.yaml 提供基础的 qlib_init 配置
- lightgbm.yaml 添加 model 相关的配置
- handler_alpha158.yaml 添加 dataset 相关的配置

合并后的 config 就包含了所有需要的设置，可以直接用来初始化实验。

## 关键设计决策

### 为什么用 Qlib 而不是自己写

你可能会想，为什么不自己从头写一套量化框架，要用现成的 Qlib？

有几个很重要的原因：

1. **成熟稳定**：Qlib 是微软开源的，经过大量实战检验，比我们自己写的更可靠。
2. **功能完整**：Qlib 提供了数据处理、因子计算、回测引擎等完整功能，自己写要花很多时间。
3. **社区支持**：Qlib 有活跃的社区，遇到问题可以找到现成的解决方案。
4. **性能优化**：Qlib 在性能上做了很多优化，处理大规模数据比自己写的更快。

这就像买车。你可以自己造车，但直接买一辆成熟的车更省心、更安全、更可靠。

### 为什么同时支持 LightGBM 和 Kronos

Big-A 支持两种模型：LightGBM 和 Kronos。

LightGBM 是一种传统的机器学习模型，基于梯度提升树。它速度快、效果好、易于理解，是量化研究中最常用的模型之一。

Kronos 是基于 Transformer 的深度学习模型。它能捕捉更复杂的时间序列模式，但需要更多的数据和计算资源。

同时支持这两种模型有几个好处：

1. **对比研究**：可以对比传统机器学习和深度学习的效果。
2. **不同场景**：有些场景适合 LightGBM，有些场景适合 Kronos。
3. **学习探索**：对于初学者来说，两种模型都值得学习和尝试。

这就像工具箱里既有锤子也有电钻。不同的任务用不同的工具，有时候还需要对比一下哪种工具更好。

### 为什么用 TopkDropout 策略

TopkDropout 是一种简单但有效的选股策略。

它的基本思路是：从股票池里选出表现最好的 k 只股票，然后随机放弃其中的一些。这样做的目的是减少对某几只股票的过度依赖。

这个策略有几个优点：

1. **简单易懂**：不需要复杂的数学公式，容易理解和实现。
2. **可解释性强**：能清楚地知道为什么选这些股票。
3. **适合 A 股**：A 股市场有自己的特点，这个策略在 A 股上表现不错。
4. **减少过拟合**：随机放弃一些股票，可以减少对历史数据的过度拟合。

想象你在招聘员工。你从所有候选人中选出最优秀的 50 个人，然后随机放弃几个。这样你就不会只依赖某几个"明星员工"，团队整体会更稳健。
